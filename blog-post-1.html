Why our stroke risk model plateaued at 0.28 PR-AUC — and what that actually means

A look at the systematic feature search behind Health Check.

When we started building Health Check, the first version of the model used all 15 available features from a public Framingham-style cardiovascular dataset. Results were middling — and the instinct was to reach for the usual fixes: rebalance the classes, tune hyperparameters, try a fancier model. We tried all three. None of it moved the needle much.

That plateau turned out to be more informative than the fixes themselves.

What we tried first

Standard playbook: SMOTE oversampling to address class imbalance, class-weighted logistic regression, and a switch to Gradient Boosting tuned on PR-AUC instead of ROC-AUC (a more honest metric when the positive class is rare — stroke cases were about 15% of the dataset).

ApproachPR-AUC
Original logistic regression0.294
SMOTE + tuned logistic regression0.268
Class-weighted logistic regression0.265
Gradient Boosting0.260

Every variant landed in the same narrow band. That's the signature of a feature ceiling, not a modeling problem — if more sophisticated models or resampling could unlock hidden signal, we'd expect to see real separation between these numbers.

The systematic search

Instead of trying more models, we ran two direct tests across all 15 features, using 5-fold cross-validated PR-AUC for stability:

  • Leave-one-feature-out: drop each feature, see whether PR-AUC gets better or worse without it
  • Univariate power: how much predictive power does each feature carry completely on its own

Five features consistently helped: age, systolic blood pressure, glucose, cigarettes per day, and sex. The other ten — including things you might expect to matter, like BMI, cholesterol, and diabetes status — added essentially nothing once those five were already in the model.

A trimmed model using only those 5 features scored 0.349 PR-AUC in cross-validation — matching or slightly beating the full 15-feature model. Two-thirds of the original feature set was dead weight for this specific target.

Why this matters more than another tuning pass

We also ran a learning curve — training the same 5-feature model on 20% up to 100% of the training data — and PR-AUC stayed flat around 0.28 the whole way. If this were a sample-size problem, more data would have helped. It didn't, which confirms the ceiling is about the features themselves, not how much of them we have.

That's a real, if modest, result: stroke risk from self-reported vitals alone has a known ceiling in the literature, and this dataset reflects that. Pushing past it would need features this dataset simply doesn't have — ECG readings, family history, prior TIA/stroke events.

What we shipped instead

Rather than keep tuning against a ceiling, we shipped the simpler 5-feature model — same performance, easier to explain, faster for users to fill in. It's live now as Health Check, with the precision limitations stated plainly in the tool itself rather than buried in a README.
